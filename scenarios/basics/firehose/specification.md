# Amazon Data Firehose Specification

This document contains a draft proposal for a Code Example for *Amazon Data Firehose Basics Scenario*, generated by the Code Examples SpecGen AI tool. The specifications describe a potential code example scenario based on research, usage data, service information, and AI-assistance. The following should be reviewed for accuracy and correctness before proceeding on to a final specification.

Amazon Data Firehose is a fully managed service for delivering real-time streaming data to destinations such as Amazon S3, Amazon Redshift, Amazon OpenSearch Service, Amazon OpenSearch Serverless, Splunk, Apache Iceberg Tables, and any custom HTTP endpoint. With Firehose, you don't need to write applications or manage resources — you configure your data producers to send data to Firehose, and it automatically delivers the data to the destination you specified. You can also configure Firehose to transform your data before delivering it.

This scenario demonstrates how to create and manage a Firehose delivery stream end-to-end: creating a stream with an S3 destination, configuring tags and server-side encryption, sending data (single records and batch), verifying stream state, listing streams, and finally cleaning up resources.

### Resources

- An Amazon S3 bucket to serve as the delivery stream destination (must be pre-created or created as part of setup)
- An IAM role that grants Firehose permissions to write to the S3 bucket

## Hello Firehose

The Hello example is a separate runnable example that demonstrates the simplest possible interaction with the Firehose service.

- Set up the Firehose service client.
- Call `ListDeliveryStreams` with a limit of 10 to list existing Firehose streams in the account.
- Display the names of the delivery streams returned.
- If `HasMoreDeliveryStreams` is true, inform the user that there are additional streams not shown.
- If no streams exist, display a message indicating that no delivery streams were found in the current region.

## Scenario

This scenario walks through the complete lifecycle of a Firehose delivery stream, demonstrating stream creation, configuration, data ingestion, verification, and cleanup.

### Setup

1. **Create a Firehose delivery stream** using `CreateDeliveryStream`
2. **Wait for the stream to become active** using `DescribeDeliveryStream`

### Configure Tags

3. **Add tags to the delivery stream** using `TagDeliveryStream`
4. **Verify tags** using `ListTagsForDeliveryStream`

### Enable Encryption

5. **Enable server-side encryption (SSE)** using `StartDeliveryStreamEncryption`

### Send Data

6. **Put a single record** using `PutRecord`
7. **Put multiple records in a batch** using `PutRecordBatch`

### Update Configuration

8. **Update the stream destination** using `UpdateDestination`

### List and Verify

9. **List delivery streams** using `ListDeliveryStreams`

### Cleanup

10. **Delete the delivery stream** using `DeleteDeliveryStream`

## Errors

| Action | Error | Handling |
| - | - | - |
| `CreateDeliveryStream` | ResourceInUseException | Notify the user that a stream with that name already exists. |
| `DescribeDeliveryStream` | ResourceNotFoundException | Inform the user that the specified stream does not exist. |
| `TagDeliveryStream` | LimitExceededException | Notify the user that the maximum number of tags has been reached. |
| `ListTagsForDeliveryStream` | ResourceNotFoundException | Inform the user that the specified stream does not exist. |
| `StartDeliveryStreamEncryption` | InvalidKMSResourceException | Notify the user that the KMS key is inaccessible. |
| `PutRecord` | ServiceUnavailableException | Retry the operation with exponential backoff. |
| `PutRecordBatch` | InvalidArgumentException | Validate the records array. |
| `UpdateDestination` | ConcurrentModificationException | Fetch the latest VersionId and retry. |
| `ListDeliveryStreams` | InvalidArgumentException | Validate the parameters. |
| `DeleteDeliveryStream` | ResourceInUseException | Wait for the stream to reach a deletable state. |
